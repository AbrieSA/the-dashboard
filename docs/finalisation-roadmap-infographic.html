Roadmap To Finalise The App

What still needs to happen before this dashboard feels truly finished

The foundation is already working: the app is live, it receives raw data, runs its own calculations, and shows clear dashboard cards. The remaining roadmap is about making the experience practical, completing the data feeds, and finishing website health reporting.

Plain-English summary

We are no longer building the engine. We are finishing the controls, plugging in the remaining data sources, and making sure the dashboard becomes something people can rely on every week.

Already working Follow Up, Google Ads, target syncing, weekly and monthly views, and app-side calculations are already live.
Next layer Improve the interface so it is easier to use in real life and remove unnecessary manual work.
Main bulk of work Finish connecting the remaining reporting sources so the dashboard reflects the whole business, not just part of it.
Final outcome A dashboard that shows what is healthy, what is drifting, what needs attention, and where action should happen next.
Phase 1

Make The App Practical

These are the improvements that make the dashboard easier to use day to day, without changing the core architecture.

UI
Optimise the interface for practical use Refine spacing, navigation, filters, hierarchy, and card layouts so the dashboard is easier to scan in real meetings.
Add editable goals directly in the app Replace spreadsheet syncing with in-app goal editing so there is one source of truth and fewer moving parts.
Outcome
Less friction for the team People can trust the dashboard and update targets without needing to jump between tools.
Phase 2

Complete The Reporting Inputs

This is the main work left. Each source needs a reliable weekly and monthly reporting flow into the app.

Sales & Funnel Reporting
Set up Salesforce weekly reporting Send raw counts for prospects, applications, processing, approved, confirmed, and arrived students.
Set up Salesforce monthly reporting Keep a dedicated monthly view so the app can compare weeks and whole months cleanly.
Traffic & Ad Reporting
Set up GA4 reporting Use GA4 for traffic and on-site activity signals that help explain lead quality and source performance.
Set up Google Ads reporting Move from test values to the real recurring ad data feed.
Set up Meta Ads reporting Add the same raw-count approach used for Google Ads so channel comparisons stay consistent.
Set up TikTok Ads reporting Bring TikTok into the same source-health structure once the raw fields are defined.

Important reporting decision

Do not assume monthly numbers should always be calculated from weekly reports.

  • If the weekly values are simple raw totals, they can often be summed into a month.
  • If the weekly values are partial snapshots, stage rates, or ratios, monthly should be reported separately or recalculated from monthly raw totals.
  • Safest approach: store raw counts and let the app calculate both weekly and monthly metrics from the right period totals.
Phase 3

Finish Website Health

This is the least defined area today, so it needs both business clarity and data plumbing.

Define What “Website Health” Means
Figure out the reporting model Decide exactly which signals matter: speed, downtime, broken links, rankings, sessions, and click-throughs.
Lock the formulas and thresholds Define what counts as healthy, drifting, or poor for each website metric.
Implement The Feed
Set up website health reporting Connect Pingdom and any SEO or analytics source needed so the dashboard has real live website-health data.
Show website health clearly in the dashboard Present site performance the same way as the other sections: what is working, what is below target, and where attention is needed.

Simple “finished” definition

  • The team can use the dashboard without needing spreadsheets or manual interpretation.
  • All major reporting sources send reliable raw data into the app on schedule.
  • The app calculates the important metrics itself.
  • The dashboard clearly highlights what is healthy, what is drifting, and what needs intervention.

Most practical order from here

  • First: improve the UI and move goals into the app.
  • Second: finish Salesforce, GA4, Google Ads, Meta Ads, and TikTok reporting.
  • Third: define and wire up website health reporting.
